# AI Usage Report  
**Project:** Chapter One – Independent Bookstore Website  
**Course:** Web Development  
**Students:** Alessandra Spino, Denise Trimm  

---

## Overview

Artificial Intelligence was used throughout the development of this project as a supporting tool to assist with structuring, debugging, and refining code. It was primarily used to supplement understanding, provide guidance, and offer suggestions, which were then reviewed, modified, and integrated manually.

All final code reflects our own implementation decisions, based on course material, in-class examples, and testing.

---

## Areas Where AI Was Used

### Wireframing (Figma and Balsamiq)

AI was used to:

- Provide guidance on structuring wireframes for key pages (e.g., homepage, book catalogue, cart page)
- Suggest general layout patterns and visual hierarchy for usability
- Offer design ideas to help differentiate page layouts and improve user flow

Initial low-fidelity wireframes were created in Balsamiq to quickly establish layout and structure. These were then refined into higher-fidelity designs in Figma, where spacing, alignment, and visual hierarchy were further developed.

All wireframes were:
- Iterated based on project requirements
- Refined to ensure consistency with the final implemented layouts


### 1. HTML Structure and Semantic Markup

AI was occasionally used to:
- Suggest semantic HTML structures (e.g., `<header>`, `<nav>`, `<main>`, `<section>`, `<article>`, `<footer>`)
- Provide examples of accessible markup, including:
  - ARIA attributes (e.g., `aria-label`)
  - Proper heading hierarchy

These suggestions were used as reference points; HTML code was subsequentlly written and adjusted manually to ensure it aligned with project requirements.

---

### 2. SCSS (Styling, Layout, and Theming)

AI was used primarily to support styling decisions and troubleshoot layout issues, rather than generate full styling solutions.

#### a. SCSS Structure and Organization
AI helped guide the transition from basic CSS to SCSS by:
- Suggesting the use of variables for colors, spacing, and typography
- Recommending a modular file structure using partials (e.g., `_variables.scss`, `_components.scss`, `_layout.scss`)

All structure and organization were implemented and maintained manually, based on project needs.

#### b. Variables and Themes
AI provided suggestions for:
- Consistent spacing variables (e.g., `$space-sm`, `$space-md`)
- Color palette organization for both light and dark modes

The initial color palette was sourced from ColorHunt.co and subsequently refined with the assistance of AI to improve accessibility and contrast. Similarly, the font pairing was selected using Fontjoy and then evaluated and adjusted with AI support to ensure both visual harmony and accessibility.

These suggestions were refined and adapted to match the project’s intended aesthetic and design direction.

#### c. Layout (Flexbox and Grid)
AI was used to:
- Troubleshoot alignment issues in card layouts and footer structure  
- Suggest grid and flexbox approaches for responsive design  

Final layouts (e.g., book cards, footer columns, cart layout) were:
- Built incrementally  
- Tested manually  
- Simplified where necessary to avoid unnecessary complexity  

#### d. Component Refinement
AI helped identify improvements for:
- Spacing and alignment in book cards  
- Consistency across repeated components  
- Visual balance between elements  

However, most styling adjustments (e.g., spacing, line-height, padding) were manually fine-tuned through testing.

---

### 3. JavaScript Support

AI was used to:
- Troubleshoot page rendering bugs and calculation discrepancies
- Refine the translation label object and language path logic through multiple iterations
- Provide guidance on dynamically rendering content in a table-based structure
- Suggest potential edge cases to improve code robustness
The final implementation of the logic was written, evaluated, adjusted and tested manually and incrementally to ensure it aligned with project requirements.

---

### 4. Content Generation and Refinement

#### a. Content Generation

AI was used to:
- Generate initial drafts of written content (e.g., descriptions, About page sections)  
- Refine tone and wording to match a professional bookstore style
- Translate written content to French  

All content was:
- Edited manually  
- Adjusted for tone and consistency  
- Reviewed to ensure it matched the project’s branding  

#### b. AI Usage Report (Documentation)

AI was used to:

- Suggest appropriate terminology and tone for academic documentation
- Help refine wording for clarity and professionalism

All content in this report was:

- Reviewed and edited to accurately reflect how AI was used throughout the project
- Adjusted to ensure it aligns with course expectations and academic integrity guidelines

---

## Modifications and Critical Evaluation

All AI-generated suggestions were:
- Reviewed for accuracy  
- Tested within the project  
- Modified to align with course requirements  

In many cases, suggestions were:
- Simplified to match the level of the course  
- Rewritten to maintain consistency with existing code  
- Adjusted to ensure full understanding before implementation  

---

## Conclusion

AI was used as a supportive and supplementary tool throughout this project. It assisted with:
- Troubleshooting issues encountered during development  
- Improving overall efficiency during the coding process  
- Providing guidance and clarification when needed  

However, AI was not relied upon to generate complete solutions. All suggestions were critically evaluated, tested, and adapted before implementation. This ensured that the final product reflects our own understanding of HTML, SCSS, and JavaScript concepts.

The use of AI contributed to a more efficient workflow while still requiring active problem-solving and decision-making. As a result, we were able to reinforce key concepts learned in class while also improving the overall quality, consistency, and usability of the final website.
